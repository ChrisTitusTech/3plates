Project overview

3plates

3plates is a contract-first, cross-platform fitness app for web, Android, and iOS. The backend owns account identity, progress, preferences, streaks, notification devices, and workout catalog data.

What Is Built

Product Surface

  • Expo app targets web, Android, and iOS from apps/mobile.
  • Sign-in, progress, preferences, notifications, and workouts screens are wired to backend APIs.
  • Offline cache and queued write behavior exist for client state flows.

Backend

  • Fastify API in apps/api serves auth, user state, preferences, progress, notifications, and workouts.
  • OAuth start/callback, account linking, session refresh, expiry, and sign-out are implemented.
  • Admin workout create, update, publish, and unpublish paths enforce authorization and concurrency checks.

Data Layer

  • Postgres and Drizzle live in packages/db.
  • User identities, sessions, progress, preferences, notification devices, streak data, and workout catalog records persist in the database.
  • Migrations are self-contained and include the database prerequisites for UUID generation.

Contracts And Quality Gates

  • packages/contract is the shared request and response source.
  • Typed error responses cover auth, payload validation, missing state, and conflict cases.
  • Root scripts run install, typecheck, API tests, contract tests, and mobile tests.
